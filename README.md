# CS250
This repository contains my coursework and project artifacts from CS-250 – Software Development Lifecycle at Southern New Hampshire University. Throughout this course, I explored Agile and Scrum methodologies while working through realistic software development scenarios based on the SNHU Travel project.

The course focused on understanding how modern development teams plan, build, test, and improve software using iterative and collaborative practices.

In CS-250, I worked to:

Apply Agile principles to real-world software development scenarios

Understand and perform Scrum roles (Product Owner, Scrum Master, Developer, Tester)

Create user stories and maintain a product backlog

Develop detailed test cases aligned to user requirements

Participate in sprint planning, daily scrums, sprint reviews, and retrospectives

Reflect on team communication and continuous improvement practices

This repository may include:

Agile Team Charter

Product Backlog with prioritized user stories

User Story documentation

Detailed test cases (initial and revised)

Agile estimation discussions

Sprint reflection journals

Final Scrum retrospective

Through this course, I strengthened the following skills:

Agile & Scrum framework application

Backlog refinement and prioritization

Writing clear and testable user stories

Test case development with measurable pass/fail criteria

Iterative planning and estimation techniques (velocity, story points)

Professional communication within Agile teams

Reflective analysis of development processes

Tools & Concepts Used

Scrum framework

Agile principles and values

Sprint planning and retrospectives

Product backlog management

Test case documentation (Excel-based templates)

Iterative and incremental development

Personal Reflection

CS-250 helped me better understand how structured collaboration improves software outcomes. Working through different Scrum roles gave me perspective on how communication, adaptability, and shared ownership directly impact project success. The experience reinforced the importance of transparency and continuous improvement in Agile environments.
